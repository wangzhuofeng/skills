# Knowledge Card Writer

## 参考模板

- 读取并遵循模板：`卡片/知识卡片.md`
- 模板中的结构与段落标题保持一致

## 生成步骤

1. 为卡片拟一个简短标题（8-20字优先），文件名使用 `YYYYMMDD-标题`（避免 `/`、`:` 等非法字符）。
2. 计算 `卡片编号`：
   - 扫描 `卡片/` 目录下已有 `卡片编号: NNN` 的最大值。
   - 新编号 = 最大值 + 1；按 3 位数字零填充（如 010）。
   - 若无法解析编号，则用日期作为编号（`YYYYMMDD`）。
3. 新建卡片文件：`卡片/<标题>.md`（若重名，追加 `-2`、`-3`）。
4. 写入 YAML frontmatter：
   - `tags` 与 `up` 保留为空
   - `Time Created` 使用当前时间
   - `卡片编号` 使用新编号
   - `categories` 列表包含 `[[知识卡片]]` 与 `[[自动生成]]`
5. 填充正文内容：
   - 首行使用 `知识卡片<编号>` 并保留标签位（如 `#主题标签/...`、`#素材标签/...`）。
   - `【原文摘录】`：放入原始片段（可含 `[[来源笔记]]` 链接）。
   - `【概念转述】`：用自己的话转述核心概念。
   - `【个人体验】`：若原文没有个人体验，写“待补充”。
   - `【行动指引】`：给出 1-3 条可执行提示；若无则写“待补充”。
6. 保持卡片长度在 300-600 字内，优先清晰与可复用。

## 质量要求

- 不做纯粘贴；必须有概念转述。
- 只保留与一个核心概念高度相关的内容。
- 需要引用来源时，使用 Obsidian 链接 `[[文件名]]`。
